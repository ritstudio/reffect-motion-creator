> **사용 방법:** 
> 이 템플릿 내용을 복사한 뒤, 하단의 **[구현할 이펙트 명세]** 영역만 원하시는 아이디어로 채워서 AI에게 전달해 주세요.

## [시스템 아키텍처 및 필수 제약조건 - AI 전용]
현재 VibeCoding 프로젝트는 일반적인 SVG 애니메이션이나 CSS/DOM 기반의 퍼블리싱 구조가 아닙니다.
새로운 이펙트 쌍(`Static 모듈`, `Anim 모듈`)을 생성할 때, 다음의 아키텍처 원리와 기술 스펙을 **절대로** 벗어나서는 안 됩니다.

1. **로고 형태 누끼 보존 (Background Skipping) [가장 중요]**
   - `<mask id="...">`나 `<clipPath>` 태그를 사용하여 원본 벡터 형태를 자르는 방식은 **절대 금지**입니다.
   - `svg-sampler.js`가 오프스크린 캔버스로부터 추출한 `sampleData.grid` (2D 배열 밝기 맵)만을 참조해야 합니다.
   - 픽셀 샘플링 결과인 `darkness = (1 - brightness)` 값이 임계치(예: `0.3`) 미만인 빈 배경 영역은 `continue;` 구문을 이용해 렌더링을 완전히 건너뛰어야 합니다. 
   - **어떤 경우에도 로고가 없는 여백이나 배경 영역까지 뻗어나가는 선이나 배경색을 칠해서는 안 됩니다. 화면 전체를 덮는 뭉개진 그래픽을 그리지 마세요.**

2. **수치 보간 우선 (Math Lerp Mapping)**
   - 도형의 크기, 굵기, 길이, 불투명도 등은 단일 고정값이 아니라, `lerp(min, max, darkness)` 함수를 통해 원본 로고의 명암에 비례하여 수학적으로 유기적인 형태를 띄어야 합니다. (원본이 검을 수록 두껍거나 크게)

3. **엔진 모듈 분리 구조 유지 (Static vs Anim)**
   - 정지 화면용(`src/engine/effects/*.js`): 애니메이션 시간 `t=0` 을 가정하여, 단일 `<svg><path>...</svg>` 문자열을 통째로 반환해야 합니다.
   - 모션 화면용(`src/animate/effects/*-anim.js`): `init()` 단계에서 Float32Array를 활용해 좌표와 값을 사전에 캐싱하고 성능을 최적화하세요. 이후 `drawFrame()` 에서 Web API인 Canvas 2D(`ctx`)를 이용해 60fps RequestAnimationFrame 으로 그려야 합니다. 화면 잔상을 막기 위해 루프 첫 줄에 반드시 `ctx.clearRect()`를 포함해야 합니다.

4. **100% Canvas API 연산 (Export 호환성 보장)**
   - MediaRecorder 또는 GIF 인코딩 시 그린스크린(`#00ff00`) 배경 합성이나 투명도 처리가 기존 루프에서 완벽히 이루어지고 있습니다. 
   - 따라서 이펙트를 그릴 때 배경색을 하드코딩하거나, `window` 또는 `document` (DOM 트리) 개입, CSS 클래스 붙이기 등 외부 우회 방법을 쓰지 마세요. 무조건 순수 렌더링 수학 연산 코드로만 기능을 구현하세요.
   - 두 모듈을 생성한 뒤에는 `src/App.jsx`의 `effectPairs` 배열 맨 뒤에 `import` 하여 안전하게 추가하세요.

## [구현할 이펙트 명세 (User Input)]
AI야, 위의 시스템 아키텍처 원칙을 철저하게 지켜서 새로운 효과 1종을 완벽하게 작성해 줘.

### 1. 이펙트 기본 정보
- **이펙트 한글명:** `[여기에 작성]` (예: ASCII 매트릭스, 도트 픽셀화 등)
- **모듈 파일명(영문):** `[여기에 작성]` (예: ascii-matrix)

### 2. 시각적 형태 및 렌더링 로직
- **형태/도형:** `[여기에 작성]` (예: 로고가 있는 곳에 + 모양 십자가를 그려 줘. 로고가 어두운 곳일수록 십자가 두께가 굵어져야 해.)

### 3. 모션 방식 (시간 변화 `t` 및 삼각함수 파장 관련)
- **모션 원리:** `[여기에 작성]` (예: 모션 탭에서 십자가들이 제자리에서 회전(rotate)하게 만들어 줘.)

### 4. 제어 파라미터 4~5가지 (UI 슬라이더 연결)
*(각 파라미터의 이름, 조절되는 내용, 최소값, 최대값, 기본값을 적어주세요)*

- `[파라미터 2]`: `[여기에 작성]`
- `[파라미터 3]`: `[여기에 작성]`
- `[파라미터 4]`: `[여기에 작성]`
